# seed-tracker

Maintain `campaign/seed-tracker.md` — the append-only log of campaign seeds. This is the workshop's memory of *what was set up where, and when it needs to pay off*.

## Seed types

- **Character** — named NPCs whose role activates later.
- **Location** — places named but not visited.
- **Fact** — things the party learns that set up future adventures.
- **Hint** — specifically for endings that have hint-gated branches (e.g., the Moon-Silver Cycle's D-inversion hints).

## Preconditions

- A campaign spine exists (`docs/campaign/<slug>.md` via `campaign-planner`).
- The seed-tracker file exists (create on first run).
- The calling context has: a seed description, type, source, target (if planting) or retrieval (if retrieving).

## File format — `campaign/seed-tracker.md`

```markdown
---
campaign: <slug>
created: <date>
last-updated: <date>
---

# Seed Tracker — <Campaign Name>

Append-only log. Seeds added in order of planting.

## Planted seeds

| Seed slug | Type | Planted in | Status | Target | Notes |
|---|---|---|---|---|---|
| mira-vaenshold-silversmith | Character | 0001 (silver-rose.md provenance) | active | #0003 | Aelwen's great-great-granddaughter |
| ... | ... | ... | ... | ... | ... |

## Retrieved seeds

| Seed slug | Retrieved in | Retrieval notes |
|---|---|---|
| ... | ... | ... |

## Hint status (campaigns with hint-gated endings)

| Hint | Planted in | Target (land at) | Status (pre-land: planted / missed / caught) |
|---|---|---|---|
| ... | ... | ... | ... |

## Orphan alerts (auto-generated by arc-audit)

<Updated by `arc-audit`. Lists planted-but-not-yet-retrieved seeds whose target adventure has played.>

## Changelog

<Append-only log of entries.>
```

## Operations

### Planting a seed

When an adventure designer or session plants a seed:

1. Check seed-tracker for an existing entry with the same slug. If exists, append "also planted in X" to its Notes; do not duplicate the row.
2. Otherwise, append a new row to "Planted seeds":
   - **Slug:** kebab-case; derive from the seed's name.
   - **Type:** one of Character / Location / Fact / Hint.
   - **Planted in:** adventure slug + file reference (e.g., `#0001 silver-rose.md Provenance`).
   - **Status:** `active` (newly planted).
   - **Target:** adventure slug (e.g., `#0003`) OR `TBD` OR `post-campaign`.
   - **Notes:** one line.

### Retrieving a seed

When an adventure or session retrieves a planted seed:

1. Find the row in "Planted seeds"; update its Status to `retrieved`.
2. Append a row to "Retrieved seeds":
   - **Slug:** same as planted.
   - **Retrieved in:** adventure slug + scene reference.
   - **Retrieval notes:** what happened (one line).

### Updating hint status

For campaigns with hint-gated endings (e.g., Moon-Silver Cycle D-hints):

1. At plant time: add to "Hint status" with status `planted`.
2. At session-log review: update status to `caught` if the session log shows the party engaged with the hint; `missed` if the hint was available but the party did not catch it.
3. Hints can be `re-available` if their recovery path (per rubric v1.2) preserves them.

### Session-handoff integration

`session-handoff` calls `seed-tracker` at its step 7 with:
- All seeds the session retrieved.
- All seeds the session planted.
- Updated hint statuses.

## Orphan detection

An **orphan seed** is a seed whose target adventure has **played** but the seed was **not retrieved**. `arc-audit` computes orphans and writes them to the "Orphan alerts" section. The DM decides per-orphan:
- **Defer** — re-target to a later adventure.
- **Drop** — mark as `dropped`; note reason.
- **Recover** — plant a recovery moment in an upcoming adventure.

## Never overwrite silently

The seed-tracker file is **append-only**. Existing rows never deleted. Status updates modify the Status column only. Any other change is flagged and the DM is asked.

## Output to user

- Seeds planted this call: count + slugs.
- Seeds retrieved this call: count + slugs.
- Hint status changes: count + entries.
- Current orphan count.

## Quality gates

- [ ] Every seed has a type.
- [ ] Every planted seed has a target (or explicit `TBD` or `post-campaign`).
- [ ] Every retrieved seed was previously planted.
- [ ] Hint statuses map to the rubric v1.2 recovery paths.
- [ ] No duplicate-slug entries.

## Anti-patterns

- Editing a retrieved-seed row to change what happened (immutable).
- Duplicating seeds across rows instead of adding "also planted in..." notes.
- Orphan seeds left active after their target adventure plays (should be defer/drop/recover).
- Planting hints without registering their recovery paths.
