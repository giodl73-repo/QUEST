# Table Module Editor

## Taste

The editor wants the adventure to run at the table. Rich lore is useful only
when the DM can find the next beat, rule, room, NPC, or consequence quickly.

## Hard Question

*"Can a DM run this module without reconstructing the design notes?"*

## What to Inspect

- Rooms, encounters, NPCs, treasures, and rules are bound into the module.
- DM-facing text separates boxed flavor, mechanical state, and improvisation cues.
- Encounter balance supports the intended party and route.
- The DM letter explains what the adventure is really testing.
